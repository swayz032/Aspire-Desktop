### Aspire Law Compliance

- [x] **Law #1 (Single Brain)** — `propertyAggregator.ts` is execution-only; it calls Adam via `fetchAdamPropertyResearch()` and combines results deterministically. No autonomous decision-making or retry loop. Adam itself is the research orchestrator; the desktop aggregator orchestrates the 3-stage pipeline per the plan.
- [x] **Law #2 (Receipts)** — Receipt written on every aggregator exit: invalid address (line 303-309), needs_correction (line 318-325), success (line 527-531), exception/catch (routes line 91-110). Four distinct coverage paths. Adam's per-provider receipts passed through via `receiptsFromAdam` and chained via `evidence[]`.
- [x] **Law #3 (Fail Closed)** — Address Validation is a hard gate: `invalid` status short-circuits with no downstream calls (verified by test). `withTimeout` resolves to `undefined` on timeout rather than crashing. All four upstream clients (`fetchAdamPropertyResearch`, `validateAddress`, `geocodeAddress`, `fetchSolarInsights`) never throw — they return structured results.
- [x] **Law #4 (Risk Tiers)** — Route is classified `GREEN` (data retrieval only, no state change beyond cache write + receipt). Migration comment classifies table as `YELLOW` for DBA review. Receipts correctly use `'SUCCEEDED'/'FAILED'/'DENIED'` status.
- [x] **Law #5 (Capability Tokens)** — `capability_token_id` is null in property receipts (pre-existing V1 architecture gap; token minting in session broker not yet implemented). Tracked as deferred item D-4.
- [x] **Law #6 (Tenant Isolation)** — RLS enforced at Supabase DB level (migration lines 49-84). `suiteId` validated at route level before aggregator call. `suiteId` forwarded to Adam (`adamResearchClient.ts` line 372) so Adam can enforce on its side. Cache read/write scoped to `WHERE suite_id = ${suiteId}::uuid`.
- [x] **Law #7 (Tools Are Hands)** — `propertyAggregator.ts` does not retry, loop, or make autonomous tool decisions. Single-pass fan-out via `Promise.allSettled`. No tool orchestrates another tool.
- [x] **Law #9 (Security & Privacy)** — Addresses truncated in all log paths. API keys not logged. Apify token absent from desktop service entirely.
