# Receptionist Sarah Message Capture Canon v2

## Purpose

Define the minimum quality bar for every call note Sarah captures.

## Required fields

Every complete call message should include:

- caller name
- callback number
- reason for the call
- urgency
- preferred callback time window
- intended recipient or target destination if known
- timestamp or associated call session reference

## Quality standard

A good message is:
- complete
- easy to skim
- specific
- free of invented details
- clear enough for the recipient to act on without replaying the whole call

## Good message example

Caller: Ricky Joy
Callback: five five five, zero one zero zero
Reason: asking for revised estimate on warehouse paint job
Urgency: wants response today if possible
Callback window: before four PM

## Bad message example

Guy called about project. Seems urgent. Call him back.

## Repetition rule

Repeat back:
- phone number
- name if unusual
- callback window if timing matters

## If the caller refuses a field

Do not force it.
Capture what is available.
Never fabricate missing detail.

## Privacy rule

Capture only what is needed for follow-up.
Do not collect payment details, banking details, or sensitive identity data.
