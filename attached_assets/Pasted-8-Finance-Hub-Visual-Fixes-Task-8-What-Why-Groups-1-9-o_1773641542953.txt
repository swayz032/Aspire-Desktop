Fix 1 — Lightswind carousel: install exact component, wire Story Mode data
File: components/finance/StoryModeCarousel.tsx, components/ThreeDCarousel.tsx

Step 1 — Install via CLI: Run npx lightswind-ui add 3d-carousel. This drops the exact Lightswind source into components/ThreeDCarousel.tsx. The @/ alias already maps to ./ in tsconfig.json so import ThreeDCarousel from '@/components/ThreeDCarousel' works with no config change.

Step 2 — Convert Tailwind to inline styles: Tailwind CSS is NOT installed in this project. After the CLI install, convert every className="..." string in the file to an equivalent inline style={{ ... }} object. All Framer Motion logic, 3D transforms, drag navigation, and auto-rotation stay exactly as Lightswind wrote them. Replace any Next.js-only imports (next/link, next/image) with plain <a> and <img>.

Step 3 — Wire Story Mode data: In StoryModeCarousel.tsx, import and use the component for the web path:

import ThreeDCarousel, { ThreeDCarouselItem } from '@/components/ThreeDCarousel';
const items: ThreeDCarouselItem[] = STORY_MODES.map(m => ({
  id: m.id,
  title: m.name,
  description: m.tagline,
  tags: m.tags ?? [],
  imageUrl: m.photo,
  accent: m.accent,
}));
Keep the Platform.OS !== 'web' early-return native fallback unchanged.

Step 4 — If CLI unavailable, fall back to manual rotor fix: Root cause of the current invisible right card: cards use left: 50% + 70% width. At offset +1 the right card's left edge lands at ~90% of the container and goes off-screen. Fix: cards use position: absolute, top: 0, left: 0, width: 100%. Transforms only handle the visual offset:

Slot	Transform	Opacity	z-index

Left (offset −1)	scale(0.95) translateX(-40%)	0.60	10
Right (offset +1)	scale(0.95) translateX(+40%)	0.60	10
Others	scale(0.9) translateX(±80%)	0	0
Card image spec — no cropping:
